# 문제 1. 짝수와 홀수

## 문제 설명

정수 N이 주어집니다. 이 수가 짝수이면 `짝수`, 홀수이면 `홀수`를 출력하세요.

## 입력 형식

- 첫째 줄에 정수 N이 주어집니다.

## 출력 형식

- N이 짝수이면 `짝수`, 홀수이면 `홀수`를 한 줄에 출력합니다.

## 예제 입력

```
4
```

## 예제 출력

```
짝수
```

## 힌트

- 짝수는 "2로 나눈 나머지가 0"인 수입니다. 나머지는 `%` 연산자로 구할 수 있어요. 즉 `N % 2`가 `0`이면 짝수입니다.
- 조건을 쓸 때는 "같으냐?"를 뜻하는 `==`를 씁니다. `N % 2 == 0` 을 소리 내어 읽으면 "N을 2로 나눈 나머지가 0과 같은가?"가 됩니다.
- 뼈대는 이렇게 생겼습니다. 빈 곳만 채워 보세요.

```csharp
if (N % 2 == 0)
{
    // "짝수" 출력
}
else
{
    // "홀수" 출력
}
```

- 글자를 그대로 출력할 때는 큰따옴표로 감쌉니다. `Console.WriteLine("짝수");` 처럼요.

### 템플릿 코드

```csharp
using System;

public class Program
{
    public static void Main()
    {
        // 정수 N을 입력받으세요.

        // N이 짝수인지 조건으로 판단하세요.
        //   - 참이면 "짝수" 출력
        //   - 거짓이면 "홀수" 출력
    }
}
```

### 테스트 케이스

| # | 입력 | 출력 |
|---|------|------|
| 1 | `4` | `짝수` |
| 2 | `7` | `홀수` |
| 3 | `0` | `짝수` |

> 참고: 0은 짝수입니다. 이런 경계값이 조건을 제대로 세웠는지 확인해 줍니다.

# 문제 2. 더 큰 수 고르기

## 문제 설명

두 정수 A와 B가 주어집니다. 두 수 중 **더 큰 수**를 출력하세요. 만약 두 수가 같다면 그 수를 그대로 출력합니다.

## 입력 형식

- 첫째 줄에 정수 A가 주어집니다.
- 둘째 줄에 정수 B가 주어집니다.

## 출력 형식

- 두 수 중 더 큰 수를 한 줄에 출력합니다. (같으면 그 값을 출력합니다.)

## 예제 입력

```
3
8
```

## 예제 출력

```
8
```

## 힌트

- "A가 B보다 큰가?"는 `A > B`로 물어봅니다.
- 뼈대는 이렇게 생겼습니다.

```csharp
if (A > B)
{
    // A가 더 크므로 A 출력
}
else
{
    // 그렇지 않으면 B 출력
}
```

- 여기서 `else`는 "A가 B보다 크지 않은 모든 경우"를 뜻합니다. 즉 A가 작을 때뿐 아니라 **두 수가 같을 때도** 이쪽으로 들어와서 B를 출력하게 됩니다. B와 A가 같은 값이니 어느 쪽을 출력해도 정답이 되지요.
- 숫자를 출력할 때는 따옴표 없이 변수 이름만 넣습니다. `Console.WriteLine(A);` 처럼요.

### 템플릿 코드

```csharp
using System;

public class Program
{
    public static void Main()
    {
        // 정수 A를 입력받으세요.

        // 정수 B를 입력받으세요.

        // A와 B를 비교하여 더 큰 수를 출력하세요.
    }
}
```

### 테스트 케이스

| # | 입력 | 출력 |
|---|------|------|
| 1 | `3`<br>`8` | `8` |
| 2 | `10`<br>`4` | `10` |
| 3 | `5`<br>`5` | `5` |

# 문제 3. 절댓값 구하기

## 문제 설명

정수 N이 주어집니다. N의 **절댓값**을 출력하세요. 절댓값이란 음수의 부호를 떼어 낸 값입니다. 예를 들어 `-5`의 절댓값은 `5`, `5`의 절댓값은 그대로 `5`입니다.

## 입력 형식

- 첫째 줄에 정수 N이 주어집니다.

## 출력 형식

- N의 절댓값을 한 줄에 출력합니다.

## 힌트

- 이번 문제의 핵심은, `if`가 **무엇을 출력할지**뿐 아니라 **어떤 값을 쓸지**까지 정할 수 있다는 점입니다.
- N이 음수일 때만 부호를 뒤집으면 됩니다. 음수 앞에 `-`를 붙이면 부호가 반대로 바뀝니다. 예를 들어 N이 `-5`라면 `-N`은 `5`가 되지요.
- 뼈대는 이렇게 생겼습니다.

```csharp
if (N < 0)
{
    // N이 음수이면, 부호를 뒤집은 값 -N 을 출력
}
else
{
    // N이 0 이상이면, N 을 그대로 출력
}
```

- N이 0일 때는 `else`로 들어가 `0`을 그대로 출력하니 걱정하지 않아도 됩니다.

### 예제 입력

```
-5
```

### 예제 출력

```
5
```

### 템플릿 코드

```csharp
using System;

public class Program
{
    public static void Main()
    {
        // 정수 N을 입력받으세요.

        // N이 음수인지 판단하세요.
        //   - 음수이면 부호를 뒤집어 출력
        //   - 아니면 그대로 출력
    }
}
```

### 테스트 케이스

| # | 입력 | 출력 |
|---|------|------|
| 1 | `-5` | `5` |
| 2 | `5` | `5` |
| 3 | `0` | `0` |

# 문제 4. 양수 · 음수 · 0

## 문제 설명

정수 N이 주어집니다. 이 수가 양수이면 `양수`, 음수이면 `음수`, 0이면 `0`을 출력하세요.

## 입력 형식

- 첫째 줄에 정수 N이 주어집니다.

## 출력 형식

- N의 상태에 따라 `양수`, `음수`, `0` 중 하나를 한 줄에 출력합니다.

## 예제 입력

```
15
```

## 예제 출력

```
양수
```

## 힌트

- 갈림길이 셋이므로 `else if`가 처음 등장합니다.
- 검사는 **위에서 아래로** 순서대로 진행되고, 참인 칸을 처음 만나면 그 칸만 실행하고 나머지는 건너뜁니다.
- 뼈대는 이렇게 생겼습니다.

```csharp
if (N > 0)
{
    // 0보다 크면 "양수"
}
else if (N < 0)
{
    // 0보다 작으면 "음수"
}
else
{
    // 위 두 경우가 모두 아니면 남는 경우는 0뿐 → "0"
}
```

- 마지막 `else`에 굳이 `N == 0` 조건을 다시 쓰지 않아도 됩니다. 양수도 음수도 아니라면 남는 값은 0밖에 없으니까요. 이렇게 "앞에서 걸러지고 남은 경우"를 `else`가 받아 주는 흐름을 느껴 보세요.

### 템플릿 코드

```csharp
using System;

public class Program
{
    public static void Main()
    {
        // 정수 N을 입력받으세요.

        // N이 양수인지, 음수인지, 0인지 세 갈래로 판단하여
        // 각각 "양수", "음수", "0"을 출력하세요.
    }
}
```

### 테스트 케이스

| # | 입력 | 출력 |
|---|------|------|
| 1 | `15` | `양수` |
| 2 | `-3` | `음수` |
| 3 | `0` | `0` |

# 문제 5. 시험 점수 등급

## 문제 설명

시험 점수(0 이상 100 이하의 정수)가 주어집니다. 아래 기준에 따라 등급을 출력하세요.

- 90점 이상 → `A`
- 80점 이상 90점 미만 → `B`
- 70점 이상 80점 미만 → `C`
- 60점 이상 70점 미만 → `D`
- 60점 미만 → `F`

## 입력 형식

- 첫째 줄에 점수(정수)가 주어집니다.

## 출력 형식

- 점수에 해당하는 등급(`A`, `B`, `C`, `D`, `F` 중 하나)을 한 줄에 출력합니다.

## 예제 입력

```
95
```

## 예제 출력

```
A
```

## 힌트

- 오늘의 마지막 문제이자 가장 생각이 필요한 문제입니다. `else if`를 여러 번 이어 붙여 다섯 갈래로 나눕니다.
- **가장 중요한 요령: 높은 점수부터 검사하세요.** 이유가 있습니다. 검사는 위에서 아래로 흐르는데, 이미 위에서 걸러진 경우는 아래로 내려오지 않기 때문입니다.
- 그래서 `90점 이상`을 먼저 걸러내고 나면, 그 아래 줄의 `80점 이상`은 사실상 "80점 이상이면서 90점 미만"을 뜻하게 됩니다. 위 칸이 90점 이상을 이미 데려갔으니까요. 덕분에 `80 <= 점수 < 90` 처럼 위아래를 모두 적을 필요 없이, 그냥 `점수 >= 80` 한쪽만 적으면 됩니다.
- 뼈대는 이렇게 생겼습니다.

```csharp
if (점수 >= 90)
{
    // "A"
}
else if (점수 >= 80)
{
    // "B"
}
else if (점수 >= 70)
{
    // "C"
}
else if (점수 >= 60)
{
    // "D"
}
else
{
    // 위 어디에도 들지 못한 나머지 → "F"
}
```

- 만약 순서를 거꾸로(낮은 점수부터) 쓰면 95점도 `점수 >= 60`에 제일 먼저 걸려 `D`가 되어 버립니다. 순서가 곧 논리인 셈이지요.

### 템플릿 코드

```csharp
using System;

public class Program
{
    public static void Main()
    {
        // 점수를 입력받으세요.

        // 높은 점수부터 차례로 검사하여
        // A / B / C / D / F 중 알맞은 등급을 출력하세요.
    }
}
```

### 테스트 케이스

| # | 입력 | 출력 |
|---|------|------|
| 1 | `95` | `A` |
| 2 | `73` | `C` |
| 3 | `45` | `F` |
